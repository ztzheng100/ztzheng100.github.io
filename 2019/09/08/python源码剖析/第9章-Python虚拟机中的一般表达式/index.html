9.1 简单内建对象的创建

  • 字节码指令将如何影响当前活动的PyFrameObject对象中的运行时栈和local名字空间（f->f_locals）
    • ++stack_pointer 指向一个内存，（0x0000也指向一个0x0000与0x0001之间的内存）++

左边的stack_pointer是运行时栈的栈顶指针，字节码指令对符号和常量的操作最终都将反应到运行时栈和local名字空间（f->f_locals）

    • DUP_TOP会获取栈顶的元素，增加其引用，又再一次将栈顶元素压入栈中，

  3. 19 ROT_TWO

    • 会取出运行时栈的第一个元素和第二个元素，然后将其位置对调

    • 将1这个对象压入到运行时栈

    • 将列表对象l写入local名字空间

9.3.1 符号搜索

  • 6 LOAD_NAME 0 (a)

    • 变量的搜索会沿着局部作用域（local名字空间）、全局作用域（global名字空间）和内建作用域（builtin名字空间）依次上溯，直至搜索成功或全部搜完3个作用域，如果找到了变量名a则压入栈中。否则，抛出异常

9.3.2 数值运算

  • 加法运算
  • BINARY_ADD
    • 加法运算，将两个参数出栈，然后相加再检测是否溢出，如果溢出则走对象相加的慢速通道，如果没有溢出则返回，如果是PyStringObject对象相加，则根据相加结果创建新的PyStringObject对象返回

9.3.3 信息输出（身体不舒服，待续）

  • 在获得输出的目标和待输出的对象后，PRINT_ITEM将通过PyFile_WriteObject->PyObject_Print->internal_print的调用序列最终调用v->ob_type->tp_print等待输出对象自身所携带的输出函数进行输出。如果对象没有定义tp_print，它就会调用tp_str或tp_repr获得对象的字符串表示形式，然后将字符串输出

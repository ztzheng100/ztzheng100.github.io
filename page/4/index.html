第3章 Python中的字符串对象

  • keyword
    *字符串对象 *
  • 可变对象和不可变对象 （list和string，tuple）（定义之后不能修改）
  • 定长对象和变长对象（int 和 list）（定义的时候）

3.1 PyStringObject与PyString_Type

  • PyStringObject对象

    • ob_size变量：保存着对象中维护的可变长度内存的大小
    • ob_sval是一个字符的字符数组，但是ob_sval实际上是作为一个字符指针指向一段内存的,(char ob_sval[1]只是存储开始第一个字符的位置，长度由ob_size记录)
    • ob_shash变量之作用是缓存该对象的hash值，这样可以避免每一次都重新计算该字符串对象的hash值。ob_shash的初始值是―1。
    • ob_sstate变量标记了该对象是否已经过intern机制的处理

  • 类型对象PyString_Type

    • tp_itemsize：指明了由变长对象保存的元素（item）的单位长度，所谓单位长度即是指一个元素在内存中的长度。这个tp_itemsize和ob_size共同决定了应该额外申请的内存之总大小是多少。
    • tp_as_number、tp_as_sequence、tp_as_mapping，三个域都被设置了。这表示PyStringObject对数值操作、序列操作和映射操作都支持。

3.2 创建PyStringObject对象

  • 创建PyStringObject对象的两种方式

  • PyString_FromString

    1. 计算长度
    2. 长度0, 空字符串, 是返回已定义好的nullstring
    3. 长度1, 字符, 返回字符缓冲池里面的
    4. 都不是, 分配内存, 初始化

  • PyString_FromStringAndSize

4.1 PyListObject对象

  • ob_item：指向元素列表所在内存块的首地址。
  • allocated:维护了当前列表中可容纳的元素的总数。
  • ob_size:实际被使用了的内存的数量则记录在ob_size

Python中的list是一个动态数组。所以，++在每一次需要申请内存时，PyListObject就会申请一大块内存++，这时申请内存的总大小记录在allocated中，而实际被使用了的内存的数量则记录在ob_size中。

2.2.1 对象创建的3种途径

  • 对象创建的3种途径
    • 因为PyInt_FromString和PyInt_ FromUnicode实际上都是先将字符串或Py_UNICODE对象转换成浮点数，然后再调用PyInt_FromFloat。

2.2.2 小整数对象

  • PyIntObject对象是不可变对象，即对象池里的每一个PyIntObject对象都能够被任意地共享。

    • 在Python 2.5(之后的也是)中，将小整数集合的范围默认设定为[-5，257）。但是你完全可以修改NSMALLPOSINTS和NSMALLNEGINTS，重新编译Python，从而将这个范围向两端伸展或收缩。对于小整数对象，Python直接将这些整数对应的PyIntObject缓存在内存中，并将其指针存放在small_ints中。

  • 小整数对象池就是一个PyIntObject指针数组(注意是指针数组), 大小=257+5=262, 范围是[-5, 257) 注意左闭右开. 即这个数组包含了262个指向PyIntObject的指针.

  • _PyInt_Init初始化创建数组

  • 2.2.3 大整数对象

    • 对于小整数，在小整数对象池中完全的缓存其pyintobject对象
    • 对其他整数，Python运行环境将提供一块内存空间，这些内存空间由这些大整数轮流使用，
    • PyIntBlock 内存块
      • 在一个PyIntBlock中维护着N_INTO_BJECTS个对象,82个
      • objects ：PyIntObject数组——objects，这就是真正用于存储被缓存的PyIntObject对象的内存
      • Python使用一个单向链表来管理全部block的objects中所有的空闲内存，这个自由内存链表的表头就是free_ list。当然，最开始的时候，这两个指针都被设置为空指针，

详解C语言中结构体的自引用和相互引用

2.2.4 添加和删除

  • 通过对PyInt_FromLong进行细致入微的考察，真实地展现一个个PyIntObject对象是如何从无到有地产生，并融入到Python的整数对象系统中的

  • PyIntObject对象的创建通过两步完成：

    • 如果小整数对象池机制被激活，则尝试使用小整数对象池；

    • 如果不能使用小整数对象池，则使用通用的整数对象池。

2.2.4.1 使用小整数对象池

  • 如果小整数对象池机制被激活(ival在【5到257】内，返回一个PyIntObject对象数组
  • 传入的值不属于小整数，就调用free_list指向的空的pyintobject的存储空间，写入数据。

2.2.4.2 创建通用整数对象池

  • 申请一块PyIntBlock大小的内存，将PyIntBlock内的objects[] 数组转换成单向链表，并返回链表的头指针（数组的最后一个元素）,返回时必须赋值给free_list 的静态指针。

  • block_list始终是指向最新创建的PyIntBlock对象。

  • 从里面拿整数时, 取free_list指向的节点, 然后free_list指向链表下一个节点

当一个block用完了之后, 即free_list=NULL, 此时要新建另一个PyIntBlock

2.2.4.3 使用通用整数对象池

  • 删除一个PyIntObject对象int_dealloc
  • 将free_list指向的下一个空PyIntObject的地址赋给要释放的PyIntObject对象，然后将free_list指向当前要释放的对象.

小整数对象池的初始化

  • 小整数对象池是实际也是使用通用整数对象池，只是整数池在程序运行过程一直有效，不会被释放 small_ints[ival + NSMALLNEGINTS]=v一直引用它。

第1章 Python对象初探

1.1 Python内的对象

  • python中所有对象共有信息的定义
  • 对象：是数据以及基于这些数据的操作的集合
  • python所有内建的类型对象都是被静态初始化的。而c结构体是在堆申请一块内存

    https://www.cnblogs.com/yechanglv/p/6941993.html

  • python中，对象A一旦被创建，它在内存中的大小就不变了。一旦对象是可变长度的数据的对象，就在对象内维护一个指向可变大小的内存区域

1.1.1 对象机制的基石__PyObject

  • _object对象

    • ob_refcnt：内存管理机制，基于引用计数的垃圾收集机制。
    • _typeobject：类型信息，后面详解。

1.1.2 定长对象和变长对象

  • ob_size：容器内所容纳元素的的个数
  • 变长对象

1.2 类型对象19

  • 在_typeobject 的定义中包含了许多的信息，主要可以分为4 类：
    • 类型名，tp_name，主要是Python 内部以及调试的时候使用；
    • 创建该类型对象时分配内存空间大小的信息，即tp_basicsize 和tp_itemsize；
    • 与该类型对象相关联的操作信息（就是诸如tp_print 这样的许多的函数指针）

1.2.1 对象的创建

  • Python 的C API 分成两类
    1. 一类称为范型的API，或者称为AOL（Abstract Object Layer）
      • 形式:PyObject_***
      • 输出对象的PyObject_Print ， 你可以PyObject_Print(int object) ， 也可以PyObject_Print(stringobject)，
    2. 另一类是与类型相关的API，或者称为COL（Concrete Object Layer）
      • 这类API 通常只能作用在某一种类型的对象上，对于每一种内建对象，Python 都提供了这样的一组API。
      • 比如对于整数对象，我们可以利用如下的API 来创建，PyObject intObj = PyInt_FromLong(10)，这样就创建了一个值为10 的整数对象。
  • int(10)的执行过程
    • 结果：当我们执行“int(10)”这样的表达式时，就是通过PyInt_Type 创建了一个整数对象。
    1. 通过PyInt_Type 创建了一个整数对象。
    2. 首先PyInt_Type 中的tp_new 会被调用，
    3. 如果这个tp_new 为NULL（真正的PyInt_Type 中并不为NULL，我们这里只是举例说明tp_new 为NULL 的情况），那么会到tp_base 指定的基类中去寻找tp_new 操作
    4. PyBaseObject_Type 的tp_new指向了object_new。

    5. 标上序号的虚线箭头代表了创建整数对象的函数调用流程，

  • typedef PyObject *(*allocfunc)(struct _typeobject *, Py_ssize_t);

    c语言函数指针的理解与使用

1.2.2 对象的行为

  • PyTypeObject 中定义了大量的函数指针，会指向某个函数/null,这些函数指针可以视为类型对象中所定义的操作，而这些操作直接决定着一个对象在运行时所表现出的行为。

  • 函数指针举例

    • tp_hash 指明对于该类型的对象，如何生成其hash 值。
    • tp_new，tp_init 是如何决定一个实例对象被创建出来并初始化的
    • tp_as_number：指向PyNumberMethods函数族，定义了数值对象应该支持的操作
      • tp_as_number.nb_add 就指定了对该对象进行加法操作时的具体行为。
    • tp_as_sequence:指向PySequenceMethods函数族，序列对象list
    • tp_as_mapping:指向PyMappingMethods函数族，关联对象dict

    2. 另一类是与类型相关的API，或者称为COL（Concrete Object Layer）
      • 这类API 通常只能作用在某一种类型的对象上，对于每一种内建对象，Python 都提供了这样的一组API。
      • 比如对于整数对象，我们可以利用如下的API 来创建，PyObject intObj = PyInt_FromLong(10)，这样就创建了一个值为10 的整数对象。

1.2.2 对象的行为

  • PyTypeObject 中定义了大量的函数指针，会指向某个函数/null,这些函数指针可以视为类型对象中所定义的操作，而这些操作直接决定着一个对象在运行时所表现出的行为。

  • 函数指针举例

    • tp_hash 指明对于该类型的对象，如何生成其hash 值。
    • tp_new，tp_init 是如何决定一个实例对象被创建出来并初始化的
    • tp_as_number：指向PyNumberMethods函数族，定义了数值对象应该支持的操作
      • tp_as_number.nb_add 就指定了对该对象进行加法操作时的具体行为。
    • tp_as_sequence:指向PySequenceMethods函数族，序列对象list
    • tp_as_mapping:指向PyMappingMethods函数族，关联对象dict
